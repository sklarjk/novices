Here we provide a list of the steps you will need to take to get PTX up and running on your local Windows machine. First, in any order:

  1. If you don't already have one you like to use, install a text editor that will produce “unformatted” text.

    If you don't know what that means, or don't already have such an editor that you like to use, we recommend installing Sublime Text, which you can download at https://www.sublimetext.com/. (You can use the trial version for free indefinitely, but will get regular pop-ups asking you to register; registration costs $70.)

    Why? You will write your PTX source documents using this editor. (For more information on text editors with respect to PTX, see [2].)

    installation difficulty level: Easy.

  2. Install Git by choosing the Windows option at https://git-scm.com/book/en/v2/Getting-Started-Installing-Git.

    Why? From the Introduction in [1]: “Git \(\ldots\) is a revision control system. What's that? It is a tool to record changes to software, a tool to experiment with changes to software, and a tool to collaborate creating software.”

    For more details on why you'll be using Git (and how to use it!), see [1].

    You will perform all of your compilations in the Git Bash shell, which is installed when you install Git. You don't need to know what it is, exactly, other than that it is a command-line interface in which you'll enter your commands. If you've used Unix or Linux, you're in your wheelhouse in the Git Bash shell. If you haven't, don't worry; we'll cover the basics of what you need to know.

    Instillation difficulty level: Easy.

  3. Sign up for a GitHub account at https://github.com.

    Why? GitHub is a code hosting platform. We will later elaborate on how we will use GitHub.

    installation difficulty level: Easy.

  4. Install xsltproc.

    We highly recommend doing this installation as described in the great Windows Getting Started video at http://mathbook.pugetsound.edu/documentation.html#getting-started-videos created by Karl-Dieter Crisman.

    Why? xsltproc is a command line tool for applying XSLT stylesheets to XML documents. You don't need to worry about what this means, but you do need to have the tool installed.

    Do not move this folder or its contents.

    Installation difficulty level: Straighforward if you follow the video instructions.
